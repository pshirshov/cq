## Deviations from the design doc (with rationale)
- **`MILESTONES_ACTIVE_GROUP_ID` value changed "M0" → "active".** The doc (§8d) specifies the on-disk header becomes the literal `## active`; the in-memory group id was previously the string `"M0"`. Keeping the id `"M0"` while emitting `## active` would desync the parser's shape assertion (which checks `group.id === MILESTONES_ACTIVE_GROUP_ID`). Setting the id to `"active"` keeps a single source of truth. No external surface exposes this id (no `enumerate_*` returns it), so the change is internal. Recorded as K-CANON-3.
- **Repo-root `docs/` ledger files are gitignored, not committed.** msunify already gitignored `docs/ledgers.yaml` + `docs/milestones.md` as dev-cwd runtime artifacts; the five new ledger files + `goals` follow the same rule. They are regenerated by `init()` on any cwd; committing them would couple the repo to a particular dev run. The brief's "fresh `docs/` gets all bootstrapped ledger files" is a runtime expectation, satisfied + tested, not a committed-fixture requirement.
- **Test fixtures use non-canonical custom ledgers** (`widgets`/`notes`/`xenos`/`alpha`). Now that defects/tasks/etc. are bootstrapped with fixed schemas + prefixes, seeding them with the suites' old ad-hoc schemas would either collide on name or diverge on schema. The dual-tests suite keeps its behavioural coverage by exercising custom ledgers with distinct prefixes; the canonical schemas get dedicated lifecycle coverage in `canonical-ledgers.test.ts`.

## Adversarial review pass (explicit, inline)

Probed the brief's five adversarial-focus points; every one held:

1. **Only bootstrap artifacts removed / tracked docs safe.** `collectArtifacts`
   derives the set from `CANONICAL_LEDGERS` (never hardcoded). `archive/<name>/`
   is in scope ONLY when the basename matches a canonical ledger AND the path is
   a directory (`fs.lstat`), so flat `docs/archive/tasks-M*.md` is excluded.
   Proven by: a unit test asserting `collectArtifacts` returns exactly
   {ledgers.yaml, per-ledger .md, archive/tasks, .locks} and NOT
   archive/tasks-M0.md or drafts/*; a backup-run test asserting seeded
   `docs/drafts/x.md` + flat `docs/archive/tasks-M0.md` SURVIVE; a flat-file
   (non-dir) `archive/tasks` exclusion test; AND a real end-to-end probe with a
   genuine `archive/tasks/` directory + a flat `archive/tasks-M5.md` —
   directory removed, flat file survived.
2. **Confirmation mandatory.** No path deletes without `y`/`--yes`. `n`/empty/
   EOF abort (exit 0, untouched). Non-TTY without `--yes` REFUSES (exit 1) —
   verified through the real CLI under `< /dev/null` with a 10s timeout that did
   NOT fire (124 would mean a hang); files untouched. `defaultConfirm` returns
   false on EOF (piped-empty probe) and true on piped `y`.
3. **Backup default + recoverable.** Default MOVE to a timestamped dir
   preserving layout; the backup tree contains every moved artifact (verified
   by path). `--no-backup` removes with no backup dir.
4. **Startup error actionable, no stack trace; non-bootstrap unaffected.**
   Wrapper test asserts the message names the ledger + `docs/<name>.md` +
   `cq reset`, contains no `at ` / `BootstrapViolationError:` stack markers, and
   that a plain `Error` propagates by identity (not reformatted). End-to-end
   boot against a staled schema printed exactly the actionable message + exit 1.
5. **Normal `cq` invocation unchanged.** `parseInvocation` delegates every
   non-`reset`-first-token argv to the byte-unchanged `parseArgs`; existing
   `args.test.ts` cases all green; a clean-cwd boot regenerated + logged
   "cq listening".

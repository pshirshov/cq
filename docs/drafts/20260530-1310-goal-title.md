# Plan — add short `title` to goals (gtitle)

Branch `goal-title` off main `7d5dcd4`. One milestone, one execution PR.

## Goal

The `goals` ledger has only `description` (required). Add a SHORT `title`
(required) for list display alongside the detailed `description` for the
expanded view. Thread end-to-end: schema → producer output → harness write →
goals.snapshot → Goals tab.

## Acceptance

- `GOALS_SCHEMA.fields` gains `title: { type: "string", required: true }`
  placed FIRST, before `description`; `description` stays required.
- `ProducerOutputSchema.goal` = `{ title: z.string().min(1), description:
  z.string().min(1) }`. `buildProducerPrompt` instructs a short title (≤ ~8
  words) + a detailed description. The CONTINUE_SPEC continuation prompt
  (`buildContinuationPrompt`) instructs that a continuation MAY refine the
  title and MUST extend the description.
- Harness `writeArtifacts` writes BOTH `title` and `description` on the new
  goal item. `writeIncrement` updates `description` (as today) AND writes
  `title` from `output.goal.title`. Append-only milestone/task invariants
  untouched.
- `GoalSnapshot` Zod (protocol.ts) gains `title: z.string()`; the snapshot
  builder (`buildGoalsSnapshot`) reads `goal.fields["title"]` into the snapshot.
- Goals tab: row header primary label = `title`; expanded body shows the
  `description` (below the title). Status chip + everything else unchanged.
- Every goal fixture / e2e mock-producer payload gains a `title`. New coverage:
  producer schema rejects missing title; harness writes title; snapshot carries
  it; Goals row renders title + expand renders description.

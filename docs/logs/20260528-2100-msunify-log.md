## Architectural decisions captured beyond the brief

1. **Tool count is 13 (8 ledger/item + 5 milestone), not the 15 the brief estimated.** Recount with explicit listing in the discharge entry.
2. **Auto-create depth-2 milestone-group on first `create_item` in a non-milestones ledger.** Brief dropped per-ledger `createMilestone` but kept `create_item(ledger, milestoneId, ...)`; the only way to materialise the group is to auto-create it. Strict-existence runs FIRST so typos still fail.
3. **`ArchiveContent` discriminated union** for `fetchArchive` covers both group archives (non-milestones) and item archives (milestones). MCP response key renamed `milestone` ‚Üí `archive` to make the discriminator visible.
4. **`now()` injection switched to ISO 8601 strings.** Tests using a deterministic counter wrap via `new Date(tick++).toISOString()` so lexicographic ordering still matches numeric (D-LED-07 monotonicity assertions preserved without arithmetic).
5. **Schema-divergence guard in `FsLedgerStore.init()`.** If a stale `milestones` entry in `ledgers.yaml` has a different schema than the library's `MILESTONES_SCHEMA`, refuse to start with `BootstrapViolationError`. Defends against out-of-band tampering.
6. **The bootstrap M0 group consumes no item-counter tick.** Init seeds an empty group; the next `createMilestone` allocates `M1` from `counters.item = 0 ‚Üí 1`.

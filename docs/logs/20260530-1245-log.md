## Review rounds / what the adversarial pass found

- **PLAN-D02 — flex-shrink defect (caught by the height-chain e2e).** The first
  fix (`.wrapper` `flex:1; min-height:0`) correctly bounded the wrapper to the
  viewport (probe: client=684, height=684), but `scrollHeight` stayed 684 — the
  wrapper is a flex column and the `.goal` rows shrank to fit instead of
  overflowing. The e2e (which tests the height chain, not the CSS property)
  failed with `scrollHeight - clientHeight = 0`. Added `.goal { flex-shrink:0 }`;
  probe then showed scroll=4719 and the spec passed. This is exactly the class
  of bug the brief's adversarial focus item (3) demanded be tested for.
- **PLAN-D01 — read-but-not-write gate.** Verified the deny set is the
  complement of `{Read,Grep,Glob} ∪ {fq submit}` (test includes a distinct-fq
  case proving only THIS phase's submit tool is allowed); verified neither
  headless lane sets `permissionMode` so the gate is actually consulted; verified
  the only MCP server is `wf` with just the submit tool ⇒ harness-owns-writes
  holds.
- **PLAN-D03 — semantic vs theme surface.** The status chips
  (clarifying/planning/done/…) keep light pastels: they encode status meaning,
  not theme surface (History-tab convention). The theme test strips the
  `.status*` rules before asserting no bare light hexes remain.

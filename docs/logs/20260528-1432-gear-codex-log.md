## Cross-cutting decisions

- **Effort enum** (shared across both platforms):
  `"none" | "low" | "medium" | "high" | "max"`.
  Claude budget tokens: none=disabled, low=4k, med=10k, high=16k,
  max=31_999 (one below SDK cap).
  Codex `modelReasoningEffort`: none="minimal", low="low", med="medium",
  high="high", max="xhigh" (real 5-tier mapping — codex-sdk's
  `ModelReasoningEffort` union includes `xhigh`, contradicting the
  brief's "saturated at high" guidance).
- **Permission-mode union**: widened in shared/protocol.ts to include
  three `codex-`-prefixed sandbox values; the per-backend bridge rejects
  values that don't match its platform.
- **Migration head**: bumped from #5 to #6 in a single migration
  bundling both new columns (session.platform + session.effort).
- **Facade routing rule**: `frame.platform ?? "claude"`. Lazy
  CodexBridge construction so tests/code paths that never touch
  Codex don't pay any import cost.
- **localStorage keys**: `cq.model`, `cq.permissionMode`,
  `cq.effort`, `cq.hideSdkEvents`. Server-side `ui_settings` table
  kept intact (D41 path); popup reads localStorage as the *primary*
  default source, and the existing `settings.get_result` handler still
  merges server-persisted values when they arrive.
- **Resume button visibility predicate** (List.tsx):
  `agentName === 'main' && endedAt !== null && sessionId !==
  activeSessionId && (currentPlatform === null ||
  currentPlatform === row.platform)`. `currentPlatform` comes from
  `modelToPlatform(localStorage.cq.model)` read in HistoryTab.

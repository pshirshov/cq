# Cycle: reset — `cq reset` subcommand + humane bootstrap-divergence startup error

Plan doc for milestone **M-RESET**. Worktree `cq-reset` off `5d0408c`.
Loop serialised (no subagent-dispatch in this harness); adversarial review
run as an explicit per-PR step.

## Problem

Changing a bootstrapped ledger's canonical schema (e.g. the recent `goals`
`title` addition) makes `FsLedgerStore.init()` throw `BootstrapViolationError`
at startup against an existing `docs/<ledger>.md` written under the old schema
(`FsLedgerStore.ts:282`). Today that surfaces as a raw stack trace from
`main.ts`. The bootstrapped ledger files are GITIGNORED, REGENERABLE
artifacts. The fix: a `cq reset` command to clear them (manual confirmation
required), plus a humane startup error pointing at it.

## Bootstrap artifact set (derived, NOT hardcoded)

From `@cq/ledger` `CANONICAL_LEDGERS` (names: milestones, defects, tasks,
hypothesis, questions, decisions, goals) plus:
- `docs/ledgers.yaml` (the registry)
- `docs/<name>.md` for each canonical ledger
- `docs/.locks/` directory (advisory lockfiles)
- `docs/archive/<name>/` directory ONLY where `<name>` exactly matches a
  canonical ledger name — NEVER flat `docs/archive/tasks-M*.md` files.

`.gitignore` already lists ledgers.yaml + per-ledger .md + `.locks/`; confirms
the set. `docs/.ledger-backup/` to be added to .gitignore.

## Architecture decision

- `cq` binary = `bun run packages/server/src/main.ts --` (flake.nix:288).
  `process.argv.slice(2)` therefore receives the user args. Subcommand
  detection lives in the arg layer: a new `parseInvocation(argv)` returns a
  discriminated union `{ kind: "server"; args: Args } | { kind: "reset";
  opts: ResetOpts }`. `main.ts` switches on `.kind`. Server path preserved
  byte-for-byte (delegates to existing `parseArgs`).
- Reset core lives in `packages/server/src/reset.ts` as a pure-ish function
  `runReset(opts, deps)` with injected `cwd`, `confirm` (async () =>
  Promise<boolean>), `now` (() => string for the backup ts), and IO sinks
  (stdout/stderr writers). Default `confirm` reads one line from stdin;
  refuses (no hang) when stdin is not a TTY and `--yes` not passed.
- The bootstrap artifact set is computed from `CANONICAL_LEDGERS` imported
  from `@cq/ledger` — single source of truth.
- Humane startup error: a small wrapper `initLedgerStoreOrExit(store, deps)`
  in a shared module (`packages/server/src/ledgerInit.ts`) that awaits
  `store.init()`, catches `BootstrapViolationError`, writes the actionable
  message to stderr and exits non-zero WITHOUT a stack trace. Re-throws all
  other errors unchanged. Called from both server.ts and devServer.ts.

## Acceptance criteria (per brief TESTS)

- reset lists + (default) BACKS UP existing bootstrap files to
  `.ledger-backup/<ts>/`; originals gone; seeded `docs/drafts/x.md`
  (tracked-style) untouched.
- `--no-backup` removes without a backup dir.
- confirmation: `n`/empty/EOF → abort, files untouched; `y` → proceed;
  `--yes` → proceed without reading stdin.
- non-TTY without `--yes` → refuses (no hang), exit non-zero.
- nothing-to-reset → clean message, exit 0.
- routing: `reset` → reset kind; normal flags → server kind.
- startup error: BootstrapViolationError caught → actionable + non-zero;
  non-bootstrap error NOT reformatted.
- regression: existing args/main/server tests green; clean-cwd server starts.

## Adversarial focus (review pass)

1. reset removes EXACTLY the bootstrap artifacts derived from constants and
   provably leaves tracked docs untouched (seed a tracked-style file + assert
   survival; assert flat docs/archive/tasks-M*.md never touched).
2. confirmation mandatory — no path deletes without `y`/`--yes`; non-TTY
   without `--yes` refuses rather than hangs.
3. backup is default + recoverable.
4. startup error actionable + no raw stack trace; non-bootstrap errors
   unaffected.
5. a normal `cq` invocation completely unchanged.

## Constraints (do NOT)

- No touching tracked docs (drafts, logs, research, archive flat files).
- No changing the divergence guard, ledger schemas, workflow/web code.
- No hang on stdin non-interactive. No sync/async unions.
- No new deps.
